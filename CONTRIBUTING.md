# Contributing to FactCheck BD

Thank you for your interest in contributing to FactCheck BD.  
This project is in the early concept stage, and contributions of all kinds are welcome, including planning, design feedback, research, and development.

## How to Contribute

### 1. Reporting Issues
If you identify a problem, unclear documentation, or a potential improvement, please open an issue.  
Include:
- A clear description of the issue  
- Steps to reproduce (if applicable)  
- Suggestions for improvement  

### 2. Suggesting Features
We welcome thoughtful feature suggestions related to:
- Fact-checking workflows  
- Verification models or methods  
- User experience (Bangla-first, mobile-first)
- Inclusion, safety, and privacy improvements  

Please open an issue titled “Feature Suggestion:” and describe the idea and rationale.

### 3. Contributing Code
As the project is in early development, core modules may not yet exist.  
However, once the prototype stage begins, contributors can:

1. Fork the repository  
2. Create a feature branch  
3. Commit changes with a clear message  
4. Submit a pull request  

Please describe the purpose of your contribution in the PR.

### 4. Documentation Contributions
Clear documentation is essential. Contributions to:
- User guides  
- System design notes  
- Data handling and safety policies  

are encouraged and can be done via pull requests or issue discussions.

## Code Style (Once Development Begins)
- Keep code clean and readable  
- Comment where necessary  
- Follow consistent naming conventions  
- Avoid adding unused dependencies  

## Community Values
We aim to build a tool focused on accessibility, safety, and public benefit.  
Please ensure contributions follow ethical and responsible AI practices.

For community safety rules, refer to the `CODE_OF_CONDUCT.md`.

Thank you for helping strengthen information integrity in Bangladesh.
